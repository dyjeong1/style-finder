## 1. 배경/문제 정의
- 최근 업로드 카드가 실제 업로드 이미지 대신 대체 썸네일을 표시한다.
- 백엔드는 현재 업로드 이미지를 실제 정적 파일로 제공하지 않고 `/mock-storage/...` 형태의 가짜 URL을 반환한다.

## 2. 목표/가설
- 1차 지표(Primary): 새로 업로드한 이미지가 최근 업로드 카드에 실제 썸네일로 표시된다.
- 2차 지표(Secondary): 기존 기록은 깨지지 않고 fallback 썸네일을 유지한다.
- 가설: 업로드 시점에 프론트에서 작은 data URL 썸네일을 만들어 localStorage에 저장하면 서버 저장소 없이도 최근 업로드 썸네일을 안정적으로 표시할 수 있다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - 업로드 파일에서 최근 업로드용 정사각 썸네일 생성
  - 업로드 히스토리에 `thumbnail_url` 저장
  - 최근 업로드 카드가 `thumbnail_url`을 우선 사용하도록 변경
  - 문서/검증 갱신
- 제외 범위:
  - 백엔드 파일 스토리지 구현
  - 기존 localStorage 기록의 실제 이미지 복원

- 기존 기록에는 실제 이미지가 없으므로 새 업로드부터 실제 썸네일이 표시된다.
