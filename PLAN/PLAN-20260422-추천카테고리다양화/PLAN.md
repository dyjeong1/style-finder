## 1. 배경/문제 정의
- 추천 기본 화면에서 네이버 쇼핑 상품이 특정 카테고리, 예를 들어 가방에 과도하게 쏠린다.
- 현재는 업로드 분석의 첫 선호 카테고리 또는 단일 카테고리 검색어만 사용한다.

## 2. 목표/가설
- 1차 지표(Primary): 전체 추천 화면에서 상의/하의/아우터/신발/가방 후보가 함께 조회된다.
- 2차 지표(Secondary): 특정 카테고리 필터를 선택하면 해당 카테고리만 조회된다.
- 가설: 전체 추천에서는 카테고리별 네이버 검색을 분산 호출하고 후보를 합치면 결과 다양성이 높아진다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - 전체 추천 시 카테고리별 네이버 검색어 생성
  - 카테고리별 분산 조회 및 중복 제거
  - 추천 응답 query 표시 개선
  - 테스트/문서 갱신
- 제외 범위:
  - 이미지에서 실제 아이템 감지 모델 도입
  - 네이버 검색 결과 후처리 ML 랭커 도입
