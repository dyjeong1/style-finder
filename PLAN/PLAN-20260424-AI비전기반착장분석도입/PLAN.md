## 1. 배경/문제 정의
- 현재 착장 분석은 색상/위치/컴포넌트 기반 규칙 비중이 높아 새로운 실사 이미지에서 일반화 한계가 있다.
- 사용자가 기대하는 수준은 이미지마다 상의/하의/아우터/가방/신발/악세서리를 새롭게 판단하는 것이다.
- 이를 위해 규칙 기반을 완전히 버리기보다, AI 비전 모델을 연결할 수 있는 분석 인터페이스와 데이터셋 구조를 먼저 정비할 필요가 있다.

## 2. 목표/가설
- 1차 지표(Primary): 비전 모델 도입 전에도 데이터셋과 인터페이스가 준비되어 실제 이미지 기반 평가 루프를 돌릴 수 있다.
- 2차 지표(Secondary): 업로드 분석 흐름에 AI 결과를 주입할 수 있는 확장 포인트가 생긴다.
- 가설: 분석 인터페이스, 데이터셋 포맷, fallback 전략을 먼저 정리하면 이후 모델 연결 작업이 작고 안전한 단위로 진행된다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - AI 비전 착장 분석 PLAN/TASK 문서화
  - 데이터셋 폴더 구조와 라벨 포맷 정의
  - 비전 분석 인터페이스 초안 및 optional fallback 경로 추가
  - OpenAI Vision provider 실제 연동과 환경변수 호환성 추가
  - Gemini 무료 티어 provider 실제 연동과 환경변수 호환성 추가
  - 테스트/문서 갱신
- 제외 범위:
  - 대규모 학습 파이프라인 구축

## 4. 일정/마일스톤
- M1(문서/구조 확정): 2026-04-24
- M2(초기 인터페이스 구현): 2026-04-24
