## 1. 배경/문제 정의
- 비즈니스 맥락: 업로드 영역에서 허용 형식을 한눈에 확인할 수 있어야 하고, 최근 업로드 목록도 직접 정리할 수 있어야 한다.
- 현재 성능/운영 이슈: 허용 형식 문구가 자리와 톤이 어색하고, 최근 업로드 카드는 쌓이기만 해서 사용자가 직접 정리할 수 없다.

## 2. 목표/가설
- 1차 지표(Primary): 업로드 placeholder 자리에 허용 형식을 자연스럽게 노출하고, 최근 업로드 카드 삭제 기능을 제공한다.
- 2차 지표(Secondary): 업로드 화면의 정보 밀도를 더 직관적으로 정리한다.
- 가설: 형식 안내 위치를 더 직관적으로 바꾸고 최근 기록 삭제를 지원하면 업로드 화면 사용성이 높아진다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위: 업로드 placeholder 문구 수정, 최근 업로드 카드 삭제 기능 추가, 문서 갱신.
- 제외 범위: 업로드 API 스키마 변경, 서버 DB 저장 구조 변경.
- 산출물(코드/모델/대시보드/문서): `frontend/app/(main)/upload/page.tsx`, `frontend/lib/api.ts`, `frontend/app/globals.css`, README/TODO, PLAN/TASK 문서.
