## 1. 배경/문제 정의
- 비즈니스 맥락: 현재 업로드 API는 이미지 파일 메타 정보만 저장하고, 추천 API는 정적인 mock 점수만 반환합니다.
- 현재 성능/운영 이슈: 업로드 이미지의 특성이 추천 결과에 반영되지 않아, 추천이 실제 분석 기반처럼 보이지 않습니다.

## 2. 목표/가설
- 1차 지표(Primary): 업로드 이미지에서 분석 요약과 feature vector를 생성하고 추천 점수에 반영
- 2차 지표(Secondary): 기존 프론트 API 호환성을 유지하면서 응답 정보 강화
- 가설: deterministic mock 분석과 벡터 점수를 도입하면 MVP 단계에서도 추천 결과의 설명력과 확장성이 좋아집니다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - 백엔드 업로드 분석 메타데이터 생성
  - 상품 mock feature vector 및 추천 점수 계산 고도화
  - 테스트 보강 및 문서 갱신
- 제외 범위:
  - 실제 ML 모델 연동
  - 외부 벡터 DB 도입

## 5. 리스크 & 가정

- 운영 가정: MVP 단계에서는 deterministic mock 분석으로도 추천 흐름 설명에 충분합니다.
