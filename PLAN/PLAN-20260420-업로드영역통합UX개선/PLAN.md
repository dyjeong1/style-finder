## 1. 배경/문제 정의
- 비즈니스 맥락: 업로드 화면에서 선택 영역과 미리보기 영역이 분리되어 있어 흐름이 한 번에 읽히지 않는다.
- 현재 성능/운영 이슈: 사용자는 업로드 박스를 누르고, 다른 위치에서 미리보기를 확인해야 해서 동선이 끊긴다.

## 2. 목표/가설
- 1차 지표(Primary): 업로드/미리보기/삭제 흐름을 하나의 영역 안에서 처리한다.
- 2차 지표(Secondary): 선택된 이미지는 정사각형 미리보기로 즉시 보여주고, 삭제 버튼으로 바로 초기화할 수 있게 한다.
- 가설: 선택-확인-삭제가 한 영역에 모이면 업로드 행동이 더 직관적으로 느껴진다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위: `/upload` 선택 영역 통합, 정사각형 미리보기, 삭제 버튼, 스타일 정리, 문서 반영.
- 제외 범위: 업로드 API 변경, 최근 업로드 데이터 구조 변경.
- 산출물(코드/모델/대시보드/문서): `frontend/app/(main)/upload/page.tsx`, `frontend/app/globals.css`, README/TODO, PLAN/TASK 문서.
