## 1. 배경/문제 정의
- 비즈니스 맥락: 추천 페이지는 실제 구매 가능한 상품 후보를 보여줘야 한다.
- 현재 성능/운영 이슈: 29CM/지그재그 실 API는 현재 프로젝트 목적과 맞지 않고, 기존 추천은 목데이터에 의존한다.

## 2. 목표/가설
- 1차 지표(Primary): 네이버 쇼핑 검색 API 인증 정보가 있을 때 실제 상품 후보를 조회한다.
- 2차 지표(Secondary): 인증 정보가 없거나 API 실패 시 기존 목데이터로 안전하게 fallback 한다.
- 가설: 네이버 쇼핑 검색 API는 검색어 기반 상품명/링크/이미지/가격 정보를 제공하므로 현재 추천 후보 조회 구조에 가장 적합하다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위: 네이버 쇼핑 검색 클라이언트, 추천 API 연동, 설정값, fallback, 테스트/문서
- 제외 범위: 제휴 딥링크/수익화, 사용자 OAuth, 실 구매 전환 추적
- 산출물(코드/모델/대시보드/문서): 백엔드 서비스 코드, 설정 문서, PLAN/TASK 문서
