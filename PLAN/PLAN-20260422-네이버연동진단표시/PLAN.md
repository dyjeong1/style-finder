## 1. 배경/문제 정의
- 네이버 쇼핑 검색 API 키를 설정했는데도 추천 페이지가 샘플 데이터로 표시된다.
- 현재 구현은 네이버 API 호출 실패 시 조용히 fallback하여 사용자가 실패 원인을 알기 어렵다.

## 2. 목표/가설
- 1차 지표(Primary): 추천 페이지에서 네이버 연동 실패 원인을 명확히 확인할 수 있다.
- 2차 지표(Secondary): 네이버 실패 시에도 기존 샘플 추천 경험은 유지한다.
- 가설: 실패 사유를 응답/화면에 노출하면 API 키 설정 문제를 빠르게 해결할 수 있다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - 네이버 쇼핑 검색 클라이언트의 실패 사유 수집
  - 추천 API 응답에 fallback 사유/메시지 추가
  - 추천 페이지에 안내 문구 표시
  - 테스트/문서 업데이트
- 제외 범위:
  - 네이버 개발자센터 앱 설정 자동 변경
  - 실제 사용자 키 저장/노출

- 인증 실패는 키 오타, 잘못된 앱 키, 검색 API 미사용 설정 등으로 발생할 수 있다.
