## 1. 배경/문제 정의
- 비즈니스 맥락: 최근 업로드 카드의 삭제 동작은 더 작고 직관적인 아이콘 형태가 자연스럽다.
- 현재 성능/운영 이슈: 삭제 버튼이 카드 바깥 아래에 있어 카드 밀도가 어색하고, placeholder 문구도 형식 표현을 더 명시적으로 맞출 필요가 있다.

## 2. 목표/가설
- 1차 지표(Primary): 최근 업로드 카드 삭제 버튼을 카드 내부 우측 상단 `x` 아이콘으로 바꾸고 placeholder 문구를 요청 문구로 맞춘다.
- 2차 지표(Secondary): 카드 정보 밀도와 업로드 안내 명확성을 함께 높인다.
- 가설: 카드 액션을 더 압축하고 형식 문구를 명시적으로 바꾸면 화면이 더 깔끔하고 이해하기 쉬워진다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위: 최근 업로드 카드 삭제 버튼 위치/스타일 수정, placeholder 문구 변경, 문서 갱신.
- 제외 범위: 업로드/추천 API 변경, 최근 업로드 저장 구조 변경.
- 산출물(코드/모델/대시보드/문서): `frontend/app/(main)/upload/page.tsx`, `frontend/app/globals.css`, README/TODO, PLAN/TASK 문서.
