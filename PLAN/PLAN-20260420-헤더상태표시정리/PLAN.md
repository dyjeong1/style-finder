## 1. 배경/문제 정의
- 비즈니스 맥락: 개인용 서비스 특성상 개발 상태 배지는 노출하지 않아도 된다.
- 현재 성능/운영 이슈: 헤더의 `Local Mode` 배지와 업로드 문구가 실제 사용 흐름에 비해 군더더기처럼 보이고, 허용 이미지 형식 안내가 화면에 드러나지 않는다.

## 2. 목표/가설
- 1차 지표(Primary): 헤더에서 `Local Mode`를 제거하고 업로드 안내 문구 및 허용 형식 안내를 더 자연스럽게 수정한다.
- 2차 지표(Secondary): 화면 톤을 더 간결하고 실제 서비스형으로 정리한다.
- 가설: 불필요한 상태 배지를 숨기고 안내 문구를 더 직관적으로 바꾸면 첫 화면 인상이 더 깔끔해진다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위: 헤더 상태 배지 제거, 업로드 카피 수정, 허용 형식 안내 추가, 관련 문서 갱신.
- 제외 범위: 헤더 구조 전면 개편, 업로드 플로우 기능 변경.
- 산출물(코드/모델/대시보드/문서): `frontend/components/app-shell.tsx`, `frontend/app/(main)/upload/page.tsx`, README/TODO, PLAN/TASK 문서.
