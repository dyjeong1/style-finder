## 1. 배경/문제 정의
- 비즈니스 맥락: 추천 결과와 위시리스트가 각각 잘 보이지만, 추천 화면에서 이미 저장한 상품인지 바로 알기 어려웠습니다.
- 현재 성능/운영 이슈: 같은 상품을 여러 번 찜하려고 시도하게 되고, 저장 상태 확인을 위해 위시리스트 페이지로 이동해야 했습니다.

## 2. 목표/가설
- 1차 지표(Primary): 추천 결과에서 저장 상태를 즉시 식별 가능
- 2차 지표(Secondary): 중복 찜 시도 감소, 추천-위시리스트 연결감 향상
- 가설: 추천 카드에 Saved 배지와 비활성화된 액션을 제공하면 흐름이 더 자연스러워집니다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - 추천 화면에서 위시리스트 저장 상태 조회
  - Saved 배지 노출
  - 저장된 상품의 버튼 상태 변경
  - 문서 및 빌드 검증
- 제외 범위:
  - 서버 API 변경
  - 위시리스트 정렬/검색 확장
