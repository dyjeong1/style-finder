## 1. 배경/문제 정의
- 비즈니스 맥락: 업로드, 추천, 위시리스트 기능은 동작하지만 전체 화면 인상이 아직 개발용 도구처럼 보여 서비스 완성도가 덜 느껴졌습니다.
- 현재 성능/운영 이슈: 기본 타이포그래피와 카드 위계가 약해 첫 화면 인상과 정보 스캔성이 아쉬웠습니다.

## 2. 목표/가설
- 1차 지표(Primary): 첫 화면과 핵심 페이지에서 더 정돈된 브랜드 인상 제공
- 2차 지표(Secondary): 업로드/추천/위시리스트 정보 스캔성 향상
- 가설: Pretendard 기반 타이포그래피와 카드/헤더 위계를 재정리하면 기능은 그대로 유지하면서도 사용 체감 품질이 올라갑니다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - Pretendard 폰트 적용
  - 공통 레이아웃/배경/네비게이션 스타일 리프레시
  - 업로드/추천/위시리스트 핵심 화면 시각 개선
  - 프론트 문서 및 루트 문서 갱신
- 제외 범위:
  - 새로운 API 추가
  - 대규모 정보구조 변경
  - 다크 모드 도입
