## 1. 배경/문제 정의
- 최근 업로드 카드에서 실제 업로드 이미지가 표시되지 않는다.
- 백엔드는 `/mock-storage/...` 형태의 이미지 URL을 반환하지만 실제 해당 파일을 서빙하지 않는다.
- 프론트는 상대경로 이미지를 프론트 서버에서 요청해 이미지 로딩에 실패할 수 있다.

## 2. 목표/가설
- 1차 지표(Primary): 새 업로드의 `image_url`을 브라우저에서 실제 이미지로 열 수 있다.
- 2차 지표(Secondary): 최근 업로드 카드가 백엔드 이미지 URL 또는 프론트 생성 썸네일을 안정적으로 표시한다.
- 가설: 백엔드가 업로드 원본 bytes를 메모리에 보관하고 `/images/{id}/file`로 반환하면 최근 업로드 이미지 표시가 안정화된다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - 업로드 원본 bytes 메모리 보관
  - 업로드 이미지 파일 응답 API 추가
  - 업로드 응답 `image_url`을 실제 라우트로 변경
  - 프론트 업로드 히스토리 이미지 URL 정규화
  - 테스트/문서 업데이트
- 제외 범위:
  - 디스크/S3 등 영구 이미지 스토리지
  - 기존 localStorage 기록의 원본 이미지 복원
