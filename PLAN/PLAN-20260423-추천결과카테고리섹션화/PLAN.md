## 1. 배경/문제 정의
- 추천 API는 여러 카테고리의 상품 후보를 반환하지만, 프론트에서는 하나의 그리드에 섞여 표시된다.
- 사용자는 전체 추천에서 상의/하의/아우터/신발/가방이 골고루 나오는지 빠르게 확인하고 싶다.

## 2. 목표/가설
- 1차 지표(Primary): 전체 추천 결과에서 카테고리별 상품군이 한눈에 구분된다.
- 2차 지표(Secondary): 기존 필터/정렬/위시리스트/상품 보기 동작을 유지한다.
- 가설: 카테고리 섹션을 추가하면 다양한 제품군 추천의 체감 품질이 높아진다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - 추천 상품을 카테고리별로 그룹핑
  - 전체 보기에서 카테고리 섹션 헤더와 섹션별 카드 그리드 표시
  - 특정 카테고리 필터 적용 시 기존 단일 그리드 흐름 유지
  - 문서/검증 갱신
- 제외 범위:
  - 추천 API 스키마 변경
  - 백엔드 추천 로직 변경
