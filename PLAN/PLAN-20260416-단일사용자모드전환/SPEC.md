## 1. 목적
개인용 로컬 서비스에 불필요한 로그인/로그아웃 의존을 제거하고, 업로드/추천/찜을 바로 사용할 수 있게 합니다.

## 2. 기능 스펙
- 백엔드는 모든 요청을 단일 로컬 사용자로 처리합니다.
- `/images/upload`, `/recommendations`, `/wishlist`는 인증 헤더 없이 동작합니다.
- 프론트 내 로그인/로그아웃 UI를 제거합니다.
- `/login` 경로는 사용자를 `/upload`로 보냅니다.

## 3. 비기능 스펙
- 기존 응답 포맷은 최대한 유지합니다.
- 위시리스트와 최근 업로드는 로컬 개발 환경에서 바로 체감 가능해야 합니다.

## 4. 구현 스펙
- 백엔드: 인증 라우터 비노출, 단일 사용자 반환, 테스트 갱신
- 프론트: 토큰 저장 제거, API 호출 단순화, E2E 시나리오 갱신
- 문서: 루트/백엔드/프론트 README와 TODO, PLAN/TASK 문서 갱신

## 5. 수용 기준
- [ ] 브라우저에서 로그인 없이 `/upload`부터 사용 가능
- [ ] `backend` 테스트 통과
- [ ] `frontend` 빌드 및 E2E 통과
