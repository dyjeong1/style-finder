## 1. 배경/문제 정의
- 비즈니스 맥락: 현재 서비스는 개인이 로컬에서 사용하는 Private 웹 서비스이며, 다중 사용자 인증이 필수 요구사항이 아닙니다.
- 현재 성능/운영 이슈: 로그인 실패가 업로드와 위시리스트 사용까지 막고 있고, 실제 필요보다 흐름이 복잡합니다.

## 2. 목표/가설
- 1차 지표(Primary): 로그인 없이 업로드, 추천, 찜 기능 사용 가능
- 2차 지표(Secondary): 로컬 개발 흐름 단순화, 안내 문구 축소
- 가설: 백엔드를 단일 사용자 고정 모드로 전환하고 프론트 인증 UI를 제거하면 개인용 서비스 운영이 더 단순해집니다.

## 3. 범위/산출물(Scope & Deliverables)
- 포함 범위:
  - 백엔드 인증 의존 제거 및 단일 사용자 고정
  - 프론트 로그인/로그아웃 UI 제거
  - E2E/문서/README/TODO 갱신
- 제외 범위:
  - 실제 회원가입 기능 추가
  - 외부 인증 제공자 연동
